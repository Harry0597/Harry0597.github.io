CTF密码学分享

定位

  • 任务？
  • 爱好？
  • 职业？

入门（大一时期）

不太推荐系统看书，一来要求高、壁垒大，二来效率低（对CTF解题而言）、可能易感觉枯燥。

相比之下，更推荐收集与学习博客性质的内容。

适合人群

  • 任何人
    • 其他方向，但有学密码需求的师傅
    • 工作中用到RSA方向的员工
    • 密码爱好者
    • 打算以此为职业

了解

  • 0xffff论坛密码指南
    文章对密码学的知识体系进行简单梳理，类似学习大纲

  • CTF-wiki-Crypto

    CTF密码学知识百科，相对较全。

    前期可以此为基础学习资料，再辅以搜索到的博客进行专题学习。

    后期充当知识目录的角色，做题时没思路了翻一翻可能有启发。

正式学习

  1. 入门、学习并尽可能熟练Python语言、Markdown语法

    目标：

    • 能基本阅读、编写Python代码
    • 能够利用Markdown记笔记，后续学习直至退役都用得上

    建议用时：2-3周

  2. 把古典极速过一遍

    目标：认识、积累代码、密码表，有编程基础的情况下，尽可能自己实现解法

    建议用时：1-2周

  3. RSA基础原理入门

    目标：以RSA为敲门砖，初步接触数论、了解概念与相关算法

    资料

    • RSA算法原理（一）
    • RSA算法原理（二）

    建议用时：1周

  4. BUUCTF 刷题

    目标:

    • 密码方向前2-3页

    • 优先解RSA相关题型，古典选择性做

    • 找找解题的感觉

    • 积累RSA模版题型

    建议：

  • 偏古典、脑洞的题目，想不到就搜wp，不要太耗费时间
  • 十几分钟完全没有思路的RSA题型，不要死磕，搜一搜WP能找到模版总结

建议用时：2-3周

  1. F2X师傅的RSA系列视频

    目标：

    • 第3步所积累的RSA题型，一来不全，二来可能一知半解。在这一步，系统学习、解决

    • 尽自己最大努力理解、记录RSA学习过程过程中所学到的数学概念、算法

    • 能看懂的全看，看完不太懂的，留个笔记、印象

    • 看完视频后，独立编写模版题型解题代码，并分明别类的收录在自己的文件夹中

    推荐用时：2-3周

  2. 收藏并反复浏览Lazzaro师傅-RSA总结

    宝藏博客。可以理解为极细化的CTF-wiki(RSA版本)

    目标：

    • 收藏
    • 像饥饿的行人看到美味、丰富的食物一样，大致浏览目录、内容
    • 不求看懂、理解，积累每个题型的特征，遇上了尽可能认得出
    • 解题、比赛没思路时浏览，有意外的收获

    推荐用时：长期

  3. 多打比赛，总结手法

  4. 锻炼搜索、理解、快速学习能力

    • 没有自己的努力，张口就问要不得
    • 搜索能力贯彻整个CTF生涯，应当结合时代，搜集当下的题型、优质博客
    • 从0到1，快速学习的能力，对非CTF领域也受益

基础（大二时期）

有了RSA学习所积累的数论基础，这个阶段要做的就是从RSA中走出来，通过专题形式尽可能广泛的学习。

适合人群

  • 其他方向，学习能力不错的师傅
  • CTF密码爱好者
  • 打算以此为职业的师傅

正式学习

  • SageMath学习

    基于Python，（数学、密码领域）高于Python；

    注意把握与Python的异同，很快就能上手

    • 建议在linux虚拟机里安装、使用
    • 资料推荐
      • USTC就读的师傅编写的SageMath简明教程
      • lazzaro师傅总结的SageMath用法
      • Vanish师傅总结的SageMath常用命令
      • 我总结的数学概念的SageMath实现

  • 基于CTF-Wiki式的专题学习

    不同于竞赛时遇到陌生题型的被动学习积累，我们平日里应主动学习，即：

    1. 视为目录，利用CTF-Wiki知道该学什么，大致了解后确定专题
    2. 综合搜索专题相关的博客、题目、题解
    3. 花一到两周进行学习，产出Markdown学习笔记

    一些必学的专题：

    • 数字签名相关
    • 椭圆曲线加密(ECC)
    • 离散对数问题(DLP)
    • 序列、分组密码
      • AES
      • LFSR
    • 随机数相关问题
      • MT19937
    • 格基密码(Lattice)
      • HNP问题
      • Coppersmith相关问题
      • …
    • 等等

  • 积极参与国内高质量比赛，同时也要大胆向国际赛迈步

进阶

进阶永无止境。

如果说，以是否掌握RSA为入门阶段的标准，以是否学会SageMath并广泛积累专题为基础阶段的分界，那么是否理解格密码与初步了解近世代数，应为进阶的门槛。

适合人群

  • CTF密码狂热爱好者
  • 打算以此为职业的师傅

升学

因现阶段选择备战考研，就业实则不太了解，也就仅谈谈关于升学。

清楚定位

回到最开始的问题，对自己的定位是什么？自己对密码学的定位是什么？

退

能看到这的师傅估计对密码都下了不少功夫，但务必清楚定位。

如果密码学是自己感兴趣、擅长的一个方向，其实是不见得要以此为职业的。

无论是出于路径依赖，觉得自己大一就在打CTF，学密码；大二也是，打的还不错；那么大三，对于升学就业的规划，默认自己就只能往纯密码方向走，其实是不合理的！

换言之，你会因为自己不抗拒物理，初中学物理，高中学物理，(假设)应试取得了较好的分数，而未来规划是一定要去当理论物理学家吗？

就业和兴趣对于大部分人来说是需要严格分割，仔细思考的。

就目前就业形势而言，如果只是一头埋进密码理论中，应是吃不饱也饿不死的。

进

那么，学习密码难道是一种错误抑或是累赘吗？

这个答案因人而异。

但在我看来，并不。

无论是走基于密码，结合其他应用场景的发展方向，还是求学读研的方向而言，学习密码的经历不会是成为负担、累赘。

相信每个师傅在抉择就业的时候，可能都想过怎么学密码似乎都找不到实习、工作？

在看我来就是定位不清晰导致的，如果实在无法忍受这一点，想着就是本科毕业即工作，其实密码学习停留到入门阶段就该收手了。

但事已至此，也无需纠结，向前看去。

  1. 若仍然坚定要搞纯理论、纯密码，那其实目标明确也挺好的。蛋糕再小也够最顶层的人吃，只需细致规划升学路线，埋头苦学即可。

    • 最好保研（保校外优质高校、研究所）

    • 考研

      • 要早些了解、开始，尽可能往高处走

      • 有自信的话，推荐报考学硕（学术型硕士）；当然，若专硕稳妥上岸，就选稳妥的

      • 推荐开始时间：国赛CISCN打完后的假期

      • 推荐考察专业课：408 / 密码相关自命题

  2. 若惊醒发现密码和自己的职业规划实在是大相径庭，我认为该及时止损，利用本科期间CTF的竞赛成果、大幅提升的提升学习能力，在大三阶段进行其他方向、领域的学习。时间不多，但尽力的话，一定来得及。

  3. 若既能接受未来从事密码理论研究，也不抗拒多领域发展。我认为以升学做缓冲很有必要。和第1类师傅相似，应以密码学基础作为敲门砖（逻辑是，密码作为专业课的院校基本不可能全院都是学密码的，你考进去自然就有学其他方向的机会）

    考研

    • 开始时间早，可以考408，院校多，选择多
    • 开始时间晚，推荐报考专业课考察密码学的院校，进行差异化竞争
    • 推荐报考专硕（专业型硕士）

最后，给一下我认为考研开始时间的早晚；

  • 极早：大二下到大三上的假期 - 考研

    • 推荐人群：基础不太好（完全没怎么学过高数、线代、概率论、408科目） 或者 目标想冲很高

  • 较早：大三上到大三下的假期 - 考研

    • 推荐人群：基础正常，目标想冲高一些

  • 正常时间：大三下开学 - 考研

    绝大多数人的开始时间，意味着没有时间优势

    • 推荐人群：目标选的比较稳妥，且某一科很有基础、自信（例如密码学、408等）

  • 较晚：大三下开学开始，比较极限

    • 推荐人群：原本没打算考研的，工作没找到合适的就极限考一次

至于具体的考研相关内容，团队每年都有考研茶话会，师傅们可以关注、了解。
